# Webマーケのデモリポジトリへようこそ
このリポジトリは、GitHubの便利な機能をシンプルにご紹介するためのデモ用プロジェクトです。

---

## 制作の流れ

### 1. ブランチを作成する
`main` ブランチから作業用ブランチを作成します。ブランチ名は担当者名やページ名などわかりやすい名前にしましょう。

```bash
git checkout -b 担当者名_ページ名
```

### 2. ファイルを編集する
HTML・CSS・画像などのファイルを編集します。作業はこのブランチ上で行います。

### 3. 変更をコミットする
編集内容をコミットして保存します。メッセージには変更内容を簡潔に書きましょう。

```bash
git add .
git commit -m "変更内容の説明"
```

### 4. GitHub にプッシュする
作業ブランチをリモートリポジトリ（GitHub）に送信します。

```bash
git push origin ブランチ名
```

### 5. プルリクエストを作成する
GitHub 上でプルリクエスト（PR）を作成し、レビューを依頼します。変更の概要や確認してほしいポイントを記載しましょう。

### 6. レビュー・修正
レビュワーがコードを確認し、コメントがあれば修正します。修正後は再度コミット＆プッシュします。

### 7. `main` にマージする
レビューが承認されたら、`main` ブランチにマージして公開します。

---

> **ブランチ運用のポイント**
> - `main` ブランチには直接コミットしない
> - 1つの作業につき1ブランチを作成する
> - こまめにコミットして変更履歴を残す
